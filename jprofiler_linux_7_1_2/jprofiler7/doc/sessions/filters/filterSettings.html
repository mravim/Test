Filter Settings

   

  The filter settings section of the session settings dialog allows you to define the filters that will be used for recording method calls. For background information on filters, please see the help topic on method call recording filters.

One exception where the filters configured in this section will not be used is if the "Disable all filters for sampling" setting is activated on the method call recording tab of the profiling settings dialog.

  The filter settings section is grouped into several tabs:
  • Define filters
    Define exclusive and inclusive filter rules for packages and classes.
  • Exceptional methods
    Configure methods whose slow invocations are shown separately in the call tree.
  • Ignored methods
    Displays methods with excessive instrumentation overhead that were removed by auto-tuning.
